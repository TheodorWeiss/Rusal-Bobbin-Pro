### _Разработано для автоматизации цеха РУСАЛ._

***О проекте***



Алгоритм системы минимизирует технологические отходы, используя авторскую модификацию метода Shelf Packing с элементами алгоритмического тетриса.
Ключевые возможности:

•	Кластеризация: Группировка заказов по марке сплава и толщине.

•	Приоритезация: Трехуровневая очередь заказов (Main, Satellites 1 & 2).

•	Умная ячейка: Автоматический каскадный поиск подходящих деталей для заполнения пустот в ряду.

•	Интерактивная визуализация: Построение чертежа раскроя каждой бобины на Canvas с отображением кромок и межкройных резов.

### *Требования к колонкам Data Mapping*

•	Номер заказа (order_id)

•	Сплав (alloy)

•	Толщина материала (мкм) (raw_thick)

•	Ширина листа заказа (мм) (width)

•	Длина листа заказа (м) (height) —  внимание: в коде ожидаются метры, которые конвертируются в мм!

•	Очередность заказа (priority) — значения 1, 2 или 3.


## *Алгоритм раскроя (ProcutterEngine)*

1.	Подготовка: Заказы разделяются на независимые группы. Для каждой группы из справочника подтягивается индивидуальный параметр inter-cut (межкройный зазор)


2.	Сортировка:

      Очередь 1: Сортировка по высоте (задают высоту ряда).

      Очередь 2 и 3: Сортировка по площади (заполняют пустоты)


3.	Укладка: Установка «якоря» (самый крупный заказ в начале ряда). 


4. Rotation: Вращение деталей на 90° (при возможности и лучшей раскладки).


3.	Каскадный поиск: Если в ряду остается место, алгоритм ищет подходящую деталь последовательно во всех очередях спутниках.


4.	Завершение: При достижении лимита длины бобины (например, 1000 м) бобина закрывается и начинается новая.

## Запуск проекта

### *Инструкция по запуску*

#### *Шаг 1: Запуск Бэкенда (Python)*
1.	В IDE (например, WebStorm) откройте терминал bash (рекомендуется назвать вкладку back)


2.	Установите зависимости (если вы не в папке backend, сначала введите cd backend):

     '''
      bash
      pip install -r requirements.txt
      Используйте код с осторожностью.
      '''

3.	Запустите сервер:


      '''
       bash
      python app.py
      Используйте код с осторожностью.
      Ожидайте строку: Running on http://127.0.0.1:5001
      '''


#### *Шаг 2: Запуск Фронтенда (React)*

1.	Откройте вторую вкладку терминала (назовите её front).
2.	Перейдите в папку проекта: 


      'cd frontend.'

3.	Установите библиотеки:


      '''
      bash
      npm install
      '''

4.	Запустите сайт:


      '''
      bash
      npm start
      '''

      Сайт откроется по адресу http://localhost:3000 (автоматически при первом пуске).

#### *Шаг 3: Тестирование*
1.	В интерфейсе выберите Excel (Заказы) и CSV (Справочник).

2.	Нажмите «Отправить».

3.	Успех: в терминале бэкенда появится код 200, а на экране — интерактивные чертежи и метрики.

## Что делать, если не работает?
•	Ошибка CORS: Проверьте наличие CORS(app) в app.py.

•	Failed to fetch: Убедитесь, что бэкенд запущен именно на порту 5001.

Терминал занят: Чтобы остановить процесс, нажмите Ctrl + C.
